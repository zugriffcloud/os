title: 'Next.js'
---

:::note
Unfortunately, Cloudflare discontinued `@cloudflare/next-on-pages`. We noticed issues with applications using `next@15.5.0` or later.
:::

After developing your application and configuring your routes to run on the edge runtime, you can use
[`@cloudflare/next-on-pages`](https://www.npmjs.com/package/@cloudflare/next-on-pages) to build your
application for both Cloudflare Pages and zugriff.

import { Steps } from '@astrojs/starlight/components';

<Steps>

1. Configure your routes and API endpoints

   ```diff lang="tsx"
   // *.(t|j)s
   + export const runtime = 'edge';

   export default function handler(
      req: NextApiRequest,
      res: NextApiResponse<{ name: string; }>
   ) {
      res.status(200).json({ name: "John Doe" });
   }
   ```

   ```diff lang="tsx"
   // *.(t|j)sx
   + export const runtime = 'experimental-edge';

   export default function Page() {
      return <h1>Hallo Welt!</h1>;
   }
   ```

2. Configure "next/image"

   ```diff lang="js"
   // next.config.js

   ...
   images: {
     ...
   +  unoptimized: true
   }
   ```

3. Build your application with [`@cloudflare/next-on-pages`](https://www.npmjs.com/package/@cloudflare/next-on-pages)

   ```zsh
    npx @cloudflare/next-on-pages
   ```

4. Deploy and Forget.

   ```zsh
    zugriff deploy
   ```

</Steps>
